2、js的历史背景

布莱登.艾奇
JavaScript诞生于1995年。起初它的主要目的是处理以前由服务器端负责的一些表单验证。在那个绝大多数用户都在使用调制解调器上网的时代，用户填写完一个表单点击提交，需要等待几十秒，完了服务器反馈给你说某个地方填错了......在当时如果能在客户端完成一些基本的验证绝对是令人兴奋的。当时走在技术革新最前沿的Netscape（网景）公司，决定着手开发一种客户端语言，用来处理这种装简单的验证。当时就职于Netscape公司的布兰登·艾奇开始着手计划将1995年2月发布的LiveScript同时在浏览器和服务器中使用。为了赶在发布日期前完成LiveScript的开发，Netscape与Sun公司成立了一个开发联盟。而此时，Netscape为了搭上媒体热炒Java的顺风车，临时把LiveScript改名为JavaScript，所以从本质上来说JavaScript和Java没什么关系。

95年     netScape（网景）  --  liveScript

java      -----  javascript   javascript和java没有关系   

h5   和  javascript  关系？

          i++  和 ++i 的异同：
           相同点：  自增变量i  都会自动加1

               如果++在前 ， 先自增 后赋值（先将自增变量的结果加1，在将该自增变量的值赋给整个表达式）

17、关系运算符

运算符 ：  >   >=   <   <=   == 等于     === 全等       !=不等于      !== 不全等
关系表达式 ：   7 < 5     6 >= 6 --  true    
关系表达式的结果是 布尔值 ：  如果关系表达式，结果是true ， 不成立 ，结果是 false

==  和   === 区别：
     ==  只比较值，不比较类型   
    ===  先比较类型，在比较值 
     10 == "10"     true        10 != "10"  false
     10  === "10"   false       10 !== "10"   true


表达式比较规范：
     1、  如果运算符两侧都是数值型  ， 正常比较
     2、  如果运算符两侧都是字符型  ， 正常比较（比较ASCII码值）   a --  97   A---65   0---48  
     3、  如果运算符两侧有一侧是纯数字字符串 ，  会将这个字符串自动转换  成数值进行比较
     4、  不能 正常比较的结果都是 false
